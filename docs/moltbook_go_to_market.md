# Vequil x Moltbook GTM Playbook

This playbook is designed for operator-led growth through OpenClaw posting and community engagement on Moltbook.

## Positioning

- Category: picks-and-shovels for the agent economy.
- Promise: see what agents actually do, catch anomalies early, and share trust publicly.
- Core proof: weekly report card with concrete numbers and anomalies.

## Posting cadence

- Monday: "Weekly agent report card" snapshot.
- Wednesday: technical teardown (anomaly detected, cost saved, workflow improved).
- Friday: user story/case study and CTA for waitlist or demo.

## Content pillars

- Transparency: "what happened" with real logs/metrics.
- Reliability: anomalies caught before cost blowups.
- Operator pride: public report card and verified quality signal.
- Ecosystem support: OpenClaw and Moltbook-first workflows.

## OpenClaw prompt template

Use this with your OpenClaw workflow for post drafts:

1) Generate 3 Moltbook post drafts for Vequil.
2) Keep each under 120 words.
3) Include one specific metric (actions, anomalies, or cost variance).
4) Include one concrete operator benefit.
5) End with CTA: "Comment 'ledger' for setup steps."
6) Tone: technical, direct, no hype words.

## Weekly KPI checklist

- New signups captured (`/api/demo`).
- Public report links shared (`/report/<event_id>`).
- Referral signups from Moltbook posts.
- Conversion from post viewers to dashboard users.

## Suggested CTA variants

- "Comment 'ledger' and I will share my OpenClaw setup."
- "Want the exact anomaly rules? Reply 'rules'."
- "I can share the Vequil plugin snippet if useful."
